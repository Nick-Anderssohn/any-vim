  <name>Task 2: Verify icon swap and vim path configuration</name>
  <what-built>
    MENU-02: Session indicator — menu bar icon changes from cursor beam to filled pencil circle during active vim session and restores on session end.
    CONF-01: Vim path configuration — menu shows current vim path, "Set Vim Path..." opens file picker, "Reset Vim Path" clears custom path.
  </what-built>
  <how-to-verify>
    **Icon Swap (MENU-02):**
    1. Look at the menu bar — you should see the AnyVim icon (cursor beam / ibeam symbol)
    2. Open a text editor (e.g., TextEdit) and type some text
    3. Double-tap the Control key to trigger AnyVim
    4. IMMEDIATELY look at the menu bar icon — it should change to a filled pencil circle
    5. In the vim window, type :wq and press Enter
    6. Look at the menu bar icon — it should return to the cursor beam symbol
    7. Repeat steps 3-6 but use :q! instead of :wq — icon should still restore
    8. (Optional) Switch between dark and light mode in System Settings > Appearance — both icons should adapt correctly

    **Vim Path Menu (CONF-01):**
    9. Click the AnyVim menu bar icon to open the dropdown menu
    10. Verify you see "Vim: (default)" as a grayed-out info item
    11. Verify you see "Set Vim Path..." as a clickable item
    12. Verify there is NO "Reset Vim Path" item (no custom path set yet)
    13. Click "Set Vim Path..." — an Open panel should appear
    14. Navigate to your vim binary (e.g., /opt/homebrew/bin/vim or /usr/bin/vim) and select it
    15. Re-open the menu — the path display should now show "Vim: /path/you/selected"
    16. Verify "Reset Vim Path" is now visible
    17. Double-tap Control to trigger — vim should work using the selected binary
    18. Click "Reset Vim Path" in the menu
    19. Re-open the menu — should show "Vim: (default)" again and "Reset Vim Path" should be gone
  </how-to-verify>
