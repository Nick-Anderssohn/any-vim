Build and run AnyVim from Xcode. Present the user with the verification steps below. This is a human-verify checkpoint — no code changes, only manual testing of the edit cycle wired in Plan 01.

What was built: Complete trigger-grab-edit-paste workflow wired in AppDelegate. On double-tap Control: captures text from any text field, opens vim in a floating terminal, and on :wq pastes the edited text back. On :q! the original text is untouched. Clipboard is preserved in all cases. Re-entrancy is guarded.

Verification steps for the user:

**Prerequisites:** Build and run AnyVim from Xcode. Ensure Accessibility and Input Monitoring permissions are granted.

**Test 1 — Happy path :wq (REST-01, REST-02):**
1. Open TextEdit. Type "Hello World" in the document.
2. Click in the text field so it's focused.
3. Double-tap Control key.
4. Vim should open in a floating window with "Hello World" loaded.
5. In vim: press `i` to enter insert mode, change text to "Hello Vim", press Esc, type `:wq` and Enter.
6. Expected: TextEdit now shows "Hello Vim" replacing the previous content.

**Test 2 — Abort path :q! (REST-03):**
1. In TextEdit with "Hello Vim" showing, double-tap Control.
2. Vim opens with "Hello Vim".
3. Type `:q!` and Enter (quit without saving).
4. Expected: TextEdit still shows "Hello Vim" — unchanged.

**Test 3 — Clipboard preservation (REST-04):**
1. Copy something distinctive to clipboard (e.g., select text in Safari and Cmd+C).
2. In TextEdit, double-tap Control.
3. Edit in vim, :wq.
4. Press Cmd+V somewhere.
5. Expected: The paste produces your original clipboard content (from step 1), not the vim-edited text.

**Test 4 — Temp file cleanup (REST-05):**
1. Before triggering, run `ls /tmp/anyvim-*` in Terminal.
2. Double-tap Control in TextEdit, edit, :wq.
3. Run `ls /tmp/anyvim-*` again.
4. Expected: No new anyvim-*.txt files remain.

**Test 5 — Re-entrancy guard (D-01):**
1. Double-tap Control in TextEdit. Vim opens.
2. While vim is open, double-tap Control again.
3. Expected: Nothing happens — no second vim window, no beep, no alert.

**Test 6 — Browser text area:**
1. Open Safari or Chrome, navigate to a page with a text area (e.g., github.com issue form).
2. Type some text, double-tap Control.
3. Edit in vim, :wq.
4. Expected: Edited text appears in the browser text area.

Resume signal: Type "approved" if all 6 tests pass, or describe which tests failed and what happened.
