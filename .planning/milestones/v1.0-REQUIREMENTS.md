# Requirements Archive: AnyVim v1.0 MVP

**Archived:** 2026-04-02
**Original Defined:** 2026-03-31
**Core Value:** Seamless vim editing in any text input on macOS — the trigger-edit-return loop must feel instant and reliable.

## v1 Requirements — All Complete

### Hotkey

- [x] **HOTKEY-01**: App detects double-tap of Control key system-wide via CGEventTap — Complete (Phase 2)
- [x] **HOTKEY-02**: Double-tap detection uses a timing threshold (~350ms) to distinguish from single taps and held modifier — Complete (Phase 2)
- [x] **HOTKEY-03**: Hotkey works regardless of which application is focused — Complete (Phase 2)

### Permissions

- [x] **PERM-01**: App checks for Accessibility permission on launch and displays guidance if not granted — Complete (Phase 1)
- [x] **PERM-02**: App checks for Input Monitoring permission on launch and displays guidance if not granted — Complete (Phase 1)
- [x] **PERM-03**: App re-checks permission state periodically and updates status accordingly — Complete (Phase 1)

### Text Capture

- [x] **CAPT-01**: On trigger, app saves the current clipboard contents (all pasteboard types) for later restoration — Complete (Phase 3)
- [x] **CAPT-02**: App sends Cmd+A, Cmd+C to the focused application via CGEvent to grab text field contents — Complete (Phase 3)
- [x] **CAPT-03**: App reads the grabbed text from the clipboard and writes it to a temporary file — Complete (Phase 3)
- [x] **CAPT-04**: App handles the case where the focused field is empty (empty temp file) — Complete (Phase 3)

### Vim Session

- [x] **VIM-01**: App opens a lightweight dedicated terminal window (not Terminal.app) with vim loaded with the temp file — Complete (Phase 4)
- [x] **VIM-02**: Terminal window uses SwiftTerm or equivalent for a PTY-backed vim session — Complete (Phase 4)
- [x] **VIM-03**: User's existing ~/.vimrc is used (vim launched normally) — Complete (Phase 4)
- [x] **VIM-04**: App detects vim process termination (user did :wq or :q!) — Complete (Phase 4)

### Text Restore

- [x] **REST-01**: On :wq, app reads the edited temp file and places contents on the clipboard — Complete (Phase 5)
- [x] **REST-02**: App sends Cmd+A, Cmd+V to the original application to replace text field contents — Complete (Phase 5)
- [x] **REST-03**: On :q!, app detects the file was not modified and skips paste-back — Complete (Phase 5)
- [x] **REST-04**: After paste-back (or abort), app restores the user's original clipboard contents — Complete (Phase 5)
- [x] **REST-05**: Temp file is deleted after the edit cycle completes — Complete (Phase 5)

### Menu Bar

- [x] **MENU-01**: App runs as a menu bar app with a status icon (no dock icon) — Complete (Phase 1)
- [x] **MENU-02**: Menu bar icon animates or changes while a vim session is active — Complete (Phase 6)
- [x] **MENU-03**: Menu includes a Quit option — Complete (Phase 1)
- [x] **MENU-04**: App launches at login (optional, configurable) — Complete (Phase 1)

### Configuration

- [x] **CONF-01**: User can configure the path to the vim binary (defaults to vim in PATH) — Complete (Phase 6)

### Compatibility

- [x] **COMPAT-01**: App works with native Cocoa text fields (TextEdit, Notes, Mail) — Complete (Phase 3)
- [x] **COMPAT-02**: App works with browser text areas (Safari, Chrome, Firefox) — Complete (Phase 3)
- [x] **COMPAT-03**: App handles timing delays between simulated keystrokes (100-150ms between Cmd+A and Cmd+C) — Complete (Phase 3)
