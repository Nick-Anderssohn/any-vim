# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-04-02
**Phases:** 1-6
**Total Plans:** 14

## Overview

AnyVim v1.0 MVP — a macOS menu bar utility that lets you use vim to edit text in any text field across any application. Double-tap Control to trigger, edit in a floating vim window, :wq to paste back. Built in 6 phases over 3 days (2026-03-31 → 2026-04-02), 9,540 lines of Swift.

## Phases

### Phase 1: App Shell and Permissions

**Goal**: A runnable background agent exists with menu bar presence and permission onboarding that does not require app restart after permissions are granted
**Depends on**: Nothing (first phase)
**Requirements**: MENU-01, MENU-03, MENU-04, PERM-01, PERM-02, PERM-03
**Plans**: 3 plans

Plans:
- [x] 01-01-PLAN.md — Xcode project, app shell, menu bar icon, Quit menu, test infrastructure
- [x] 01-02-PLAN.md — Permission checking, re-polling, sequential alerts, login item, notifications
- [x] 01-03-PLAN.md — Manual smoke test of full Phase 1 flow

**Success Criteria:**
1. App runs in the menu bar with no Dock icon and persists after the launch terminal closes
2. App shows actionable guidance when Accessibility or Input Monitoring permission is missing
3. After the user grants a missing permission in System Settings, the app detects it within a few seconds without requiring a restart
4. Menu contains a functioning Quit item that terminates the process cleanly
5. App can be configured to launch at login

### Phase 2: Global Hotkey Detection

**Goal**: Double-tapping Control triggers the app reliably from any focused application without false-positives on normal keyboard use
**Depends on**: Phase 1
**Requirements**: HOTKEY-01, HOTKEY-02, HOTKEY-03
**Plans**: 2 plans

Plans:
- [x] 02-01-PLAN.md — HotkeyManager with CGEventTap, double-tap state machine, tap health monitor, and unit tests
- [x] 02-02-PLAN.md — Wire HotkeyManager into AppDelegate and MenuBarController, manual verification

**Success Criteria:**
1. Double-tapping Control within ~350ms fires the trigger while the user is in any application
2. Single Control taps, held Control, and Ctrl+other-key combinations do not trigger the app
3. The event tap continues functioning after extended running (no silent tap death)

### Phase 3: Accessibility Bridge and Clipboard

**Goal**: The app can grab text from any focused text field and paste edited text back, leaving the user's clipboard exactly as it was before the trigger
**Depends on**: Phase 2
**Requirements**: CAPT-01, CAPT-02, CAPT-03, CAPT-04, COMPAT-01, COMPAT-02, COMPAT-03
**Plans**: 3 plans

Plans:
- [x] 03-01-PLAN.md — System API protocols, ClipboardGuard (clipboard snapshot/restore), TempFileManager
- [x] 03-02-PLAN.md — AccessibilityBridge (capture/restore via Cmd+A/C/V), wire into AppDelegate
- [x] 03-03-PLAN.md — Manual verification in native apps and browsers

**Success Criteria:**
1. Triggering in a populated text field captures the existing text correctly in native Cocoa apps and browser text areas
2. Triggering in an empty text field produces an empty temp file without crashing or hanging
3. After the edit cycle completes, the user's original clipboard contents are identical to before
4. The simulated keystrokes do not race — text is reliably captured and pasted back

### Phase 4: Vim Session

**Goal**: A floating terminal window opens with the user's text loaded in vim, and the app reliably detects when the user exits via :wq or :q!
**Depends on**: Phase 1
**Requirements**: VIM-01, VIM-02, VIM-03, VIM-04
**Plans**: 2 plans

Plans:
- [x] 04-01-PLAN.md — SwiftTerm dependency, VimPanel, VimSessionManager, protocols, and unit tests
- [x] 04-02-PLAN.md — Wire VimSessionManager into AppDelegate, manual verification

**Success Criteria:**
1. A dedicated floating terminal window opens with vim and the temp file loaded, without launching Terminal.app
2. The user's ~/.vimrc settings are applied in the vim session
3. Closing vim with :wq signals the app that an edit was completed
4. Closing vim with :q! signals the app that the edit was aborted

### Phase 5: Edit Cycle Integration

**Goal**: The complete trigger-grab-edit-paste workflow functions end-to-end, handling the happy path and all abort/error paths with temp file cleanup on every exit
**Depends on**: Phases 2, 3, 4
**Requirements**: REST-01, REST-02, REST-03, REST-04, REST-05
**Plans**: 2 plans

Plans:
- [x] 05-01-PLAN.md — Testability protocols, edit cycle wiring in AppDelegate, unit tests for all REST requirements
- [x] 05-02-PLAN.md — Manual end-to-end verification in TextEdit and browser

**Success Criteria:**
1. After :wq, the edited text appears in the original text field replacing the previous contents
2. After :q!, the original text field is unchanged
3. The temp file is deleted after every edit cycle, regardless of how vim exited
4. Triggering the hotkey while an edit session is already active does nothing

### Phase 6: Polish and Configuration

**Goal**: The app provides visual feedback during active sessions and lets the user point to a non-default vim binary
**Depends on**: Phase 5

## Milestone Summary

**Key Decisions:**
- Swift over Go — all core APIs (CGEventTap, AXUIElement, NSStatusItem) are native Swift
- No-storyboard menu bar app — pure code setup via `static func main()`
- Developer signing required from Phase 1 — TCC grants are code-identity-dependent
- Temp file for vim I/O — vim naturally writes to files on :wq
- SwiftTerm for vim hosting — embedded terminal emulator with proper PTY
- Mtime-based exit detection — compare file modification date before/after vim runs
- NSPanel with hidesOnDeactivate=false — floating panel stays visible when clicking other apps
- Strip trailing newline after :wq — vim always appends trailing newline on save
- Composing resolver for vim path — UserDefaultsVimPathResolver wraps ShellVimPathResolver

**Issues Resolved:**
- hidesOnDeactivate defaults to true on NSPanel — fixed with explicit `= false`
- Trailing newline added by vim on every :wq — stripped with regex before paste-back
- ShellVimPathResolver blocks main thread 100-300ms — static menu label, resolve at trigger time

**Technical Debt Incurred:**
- HOTKEY-02 checkbox stale in REQUIREMENTS.md (implementation complete)
- TempFileManager() instantiated fresh in .saved path rather than reusing injected instance
- 5/6 phases have draft Nyquist VALIDATION.md files never fully executed
