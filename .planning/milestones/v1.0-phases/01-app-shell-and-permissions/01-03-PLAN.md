  <action>
Build and launch the app for manual verification. Run: `xcodebuild build -scheme AnyVim -destination 'platform=macOS'` then open the built .app bundle.

Present the following smoke test checklist to the user:

**1. Menu Bar Presence (MENU-01)**
- Launch the app
- Verify: character.cursor.ibeam icon appears in the menu bar
- Verify: NO icon appears in the Dock
- Verify: App does NOT appear in Force Quit dialog (Cmd+Opt+Esc)

**2. Permission Alert Flow (PERM-01, PERM-02)**
- On first launch (or after clearing UserDefaults), an alert should appear:
  - Title: "Accessibility Permission Required"
  - Body includes: "AnyVim needs Accessibility access..."
  - Body includes: "AnyVim will also launch at login automatically..."
  - Buttons: "Open System Settings" and "Later"
- Click "Open System Settings" — System Settings should open to Privacy and Security > Accessibility

**3. Permission Re-Detection (PERM-03)**
- Grant Accessibility permission in System Settings
- Verify: Within ~3 seconds, a macOS notification appears: "Accessibility permission granted."
- Verify: A second alert appears for Input Monitoring
- Grant Input Monitoring permission
- Verify: Notification appears: "Input Monitoring permission granted."

**4. Menu Dropdown (D-03, D-06)**
- Click the menu bar icon
- Verify menu items in order:
  - "Accessibility: Granted" (non-clickable if granted)
  - "Input Monitoring: Granted" (non-clickable if granted)
  - Separator
  - "Launch at Login" with checkmark (enabled by default per D-07)
  - Separator
  - "Quit AnyVim" with Cmd+Q shortcut

**5. Launch at Login Toggle (MENU-04, D-09)**
- Click "Launch at Login" to uncheck it
- Open menu again — checkmark should be gone
- Click again to re-enable — checkmark should return

**6. Quit (MENU-03)**
- Click "Quit AnyVim"
- Verify: App terminates cleanly (menu bar icon disappears, process exits)
